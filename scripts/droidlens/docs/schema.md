# DroidLens Data Schemas

This document describes the stable data files used by `scripts/droidlens`.

## Project Profile

Path:

```text
<project>/.droidlens/profile.json
```

Purpose:

- Stores project-specific app resolution hints.
- Can be committed when it contains no machine-local paths or secrets.
- Should not contain device serials, SDK paths, or user-specific state.

Current schema:

```json
{
  "schemaVersion": 1,
  "projectId": "example",
  "app": {
    "variant": "debug",
    "package": "com.example.app",
    "activity": "com.example.app.MainActivity",
    "component": "com.example.app/com.example.app.MainActivity"
  }
}
```

Compatibility rules:

- Unknown fields must be ignored.
- `component` is preferred when present.
- `package` plus `activity` can be used to build `component`.
- When profile data is stale, tools should fail with a structured error or fall back to explicit `--app`.

## Generated Environment

Path:

```text
<project>/.droidlens/env.sh
```

Purpose:

- Generated by `doctor.sh --ensure`.
- Stores machine-local values such as ADB path and selected serial.
- Must not be committed.

Current format:

```bash
: "${DROIDLENS_ADB:=/path/to/adb}"
: "${ANDROID_HOME:=/path/to/sdk}"
: "${DROIDLENS_SERIAL:=device-serial}"
: "${DROIDLENS_APP:=auto}"
```

Compatibility rules:

- Values use shell `${VAR:=value}` so explicit environment variables win.
- Tools should source this file through `lib.sh`, not directly.

## Dangerous Action Policy

Path:

```text
<project>/.droidlens/policy.json
<project>/.droidlens/audit.jsonl
```

Purpose:

- Stores short-lived local grants for dangerous actions during AFK runs.
- Must not be committed.
- Audit log records grant, allow, deny, and revoke events.

Current policy schema:

```json
{
  "schemaVersion": 1,
  "grants": [
    {
      "id": "6f2d9a3b1c00",
      "action": "clear-app-data",
      "app": "com.example.app",
      "serial": "device-serial",
      "createdAt": 1779120000,
      "expiresAt": 1779123600,
      "ttlSeconds": 3600,
      "maxRuns": 1,
      "usedRuns": 0,
      "reason": "AFK regression reset",
      "createdBy": "user"
    }
  ]
}
```

Matching rules:

- `action` must match exactly.
- `app` must match the resolved package; `*` is allowed but should be rare.
- `serial` must match the selected device; `any` / `*` allow any device.
- A grant is active only before `expiresAt`, before `maxRuns` is exhausted, and before `revokedAt` is set.
- Consuming a grant increments `usedRuns` atomically and appends an audit event.

## Page Tree

Path:

```text
~/.droidlens/page-tree.json
```

Purpose:

- Stores learned pages and navigation edges across sessions.
- User/device memory, not project source.

Current schema:

```json
{
  "version": 1,
  "buckets": {
    "com.example.app@42+1080x2340@440": {
      "pages": {
        "Home": {
          "title": "Home",
          "activity": "com.example.app.MainActivity",
          "fingerprint": "abc123",
          "texts": ["Home", "Settings"],
          "buttons": {
            "text:Settings": {
              "x": 100,
              "y": 200,
              "via": "text",
              "value": "Settings"
            }
          }
        }
      },
      "edges": [
        {
          "from": "Home",
          "via": "text:Settings",
          "to": "Settings",
          "dynamic": false,
          "status": "active",
          "failCount": 0
        }
      ]
    }
  }
}
```

Bucket key:

```text
<package>@<versionCode>+<width>x<height>@<density>
```

Compatibility rules:

- `version` must be checked before future migrations.
- Unknown page/edge fields must be preserved.
- Writes must go through `pagetree.py` so locking and atomic save are applied.
- Dynamic edges do not cache coordinates and must be re-resolved from XML selectors.

## Screenshot Metadata

Path:

```text
<screenshot-stem>.meta.json
```

Purpose:

- Maps compressed image coordinates back to ADB device coordinates.

Current schema:

```json
{
  "image": "/tmp/shot.webp",
  "mode": "--thumb",
  "coordinateSystem": "device pixels from adb screencap/uiautomator bounds",
  "deviceWidth": 1080,
  "deviceHeight": 2340,
  "imageWidth": 360,
  "imageHeight": 780,
  "scaleX": 3.0,
  "scaleY": 3.0
}
```

Compatibility rules:

- `tap.sh --meta` must reject stale metadata when current device size differs.
- Coordinate conversion must use the sidecar dimensions, not hardcoded screen sizes.
